Описание БД Магазины/Мерчендайзинг
1) Shops - описание работающих  магазинов
	Id - Уникальный идентификатор
	Market_name - Название ТЦ
	Square - площадь ТЦ
	City - Город
2) Visits - описание визитов мерчендайзеров
	Id - Уникальный идентификатор
	Product_id - id продукта
	Visit_date - Дата посещения
	Line_size - Линейный размер (длина выкладки на витрине (Base_size*count(выкладки а не всех в магазине)))
	Employer_id - id сотрудника
	Shop_id -id магазина
3) Product - описание товаров (в каждом магазине в день ревизии)
	Id - Уникальный идентификатор
	Product_name - название товара
	Shop_id -id магазина
	Revizion_date - день ревизии
	Count - число товаров
	Price_sale_out - цена отпускная(та что на ценнике в магазине)
	Base_size - линейный размер 1 единицы товара
	
(Count,Price, Base_size ) действуют со дня ревизии(включительно) по день следующей ревизии (не включительно)

4) Employers - характеристики сотрудников
	Id - Уникальный идентификатор
	Name - Имя сотрудника
	Shop_id - id магазина
	Salary - Зарплата(на руки)
	Years_exp - опыт работы в годах
	Department - департамент(Мерчендайзинг, HR, IT, Финансы)
5) NSI_market_size - описание соответствия (  размера -  тип торговой точки )
	Size_name - тип магазина (Ларек,  минимаркет, магазин у дома,  супермаркет,  гипермаркет)
	Min_square - минимальный размер для того чтобы присвоить текущую категорию
6) NSI_требования к торговой точке
	Size_name - тип магазина (Ларек,  минимаркет, магазин у дома,  супермаркет,  гипермаркет)
	Count_employers - число требуемых  сотрудников
	Base_cost - минимальная зп сотруднику

7) Product_deliv - поставки продуктов
	Product_id -id продукта
	Vendor_id - id поставщика
	Shop_id - id магазина
	Delivery_day - день поставки в магазин
	Price_sale_in -  цена от посдавщика
	Count - число поставленного товара
	
8) Vendors -  описание постащика
	id -уникальный идентификатор
	Name - название поставщика
	Country - страна
	Employers_count - число сотрудников
